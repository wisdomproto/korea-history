01 컨셉 — 노트와 블로그의 2-track 전략

사용자 의도를 기준으로 두 종류의 콘텐츠를 명확히 분리. 같은 주제라도 검색 의도가 다른 두 청중에 모두 도달.

구분요약노트 (/notes)블로그 (/blog)
사용자 의도"빠른 복습", "시험 직전""이해하고 싶다", "처음 공부"
검색 의도 매치"한능검 [시대] 정리""병자호란 원인", "훈민정음 창제 과정"
분량약 2,300자5,000~8,000자
구조표·요약 중심인트로 → 섹션 → 비교표 → FAQ → CTA
SEO 강도중간 (얕음)강함 (의미적 깊이)
JSON-LDArticle + VideoObjectBlogPosting + Breadcrumb + FAQPage
왜 분리하는가

같은 주제를 한 페이지에 다 넣으면 너무 길고, 빠른 복습용으로는 부적합. 두 청중을 분리하면 cannibalization도 막고 각자의 검색 의도에 정확히 매치.

또한 글에 끝부분에서 노트로 cross-link, 노트 상단에서 블로그로 cross-link → 사이트 내부 링크가 풍부해져 SEO 시그널 ↑.

04 글 1편 구조 — 8요소

22편 모두 동일 템플릿. 사용자 학습 효율 + SEO 친화 둘 다 최적화.

1. 인트로 후크

약 300자. 사용자가 검색해서 들어왔을 때 즉시 흥미를 끄는 문장. 핵심 인물·사건·날짜 강조.

2. TL;DR aside

5~6개 핵심 항목 bullet. 빠른 정리용 (글 다 안 읽고 핵심만 보려는 사용자용).

3. 배경 H2

1,200~1,500자. 왜 이 사건/제도가 등장했는지 큰 흐름.

4. 전개 / 진행 H2

1,200~1,500자. 시간 순으로 어떻게 진행됐는지.

5. 결과·영향 H2

1,000~1,200자. 정치·경제·사회·문화 등 영역별 결과.

6. 비교표 또는 인물표

비교 (ex. 대동법 vs 균역법) 또는 인물 정리 표. SEO 친화 + 사용자 가독성.

7. 한능검 출제 패턴 H2

5가지 빈출 유형 정리. 시험 대비 실용 가치 + "한능검" 키워드 자연 노출.

8. 마무리 + FAQ

5개 FAQ Q&A (FAQPage 스키마 = Google rich result 노림). 다음 학습 가이드.

SEO 시그널 자동 주입

  • JSON-LD: BlogPosting + BreadcrumbList + FAQPage 3종 자동 주입
  • Title: primary keyword 앞에 surfacing (예: "병자호란 원인부터 결과까지")
  • Description: excerpt에 primary + 핵심 키워드 자연 포함
  • Keywords: 12~14개 매핑 (primary + 변형 + 인물 + 시대)
  • about 스키마: primary keyword + 시대 + "한국사능력검정시험"

05 예약 발행 인프라 — 자동 라이브

DB 없이 JSON 파일 + 날짜 필터 + 매일 cron 재빌드로 구성. 운영 비용 0, 단순함 최대화.

3-step 흐름

단계위치역할
1. 콘텐츠 저장data/blog/{slug}.jsonpublishedAt 필드로 발행 날짜 지정
2. 가시성 필터web/lib/blog.ts isLive()PROD에서 publishedAt > KST today 자동 숨김. DEV는 모두 노출.
3. 자동 재빌드cron.service.ts startBlogRebuildCron()매일 09:00 KST VERCEL_DEPLOY_HOOK_URL 호출 → Vercel 재빌드 → 그날자 글 자동 라이브

장점

  • DB 0: JSON 파일이 git에 있어 version 관리 자동
  • SSG 빠름: 빌드 시점에 결정되어 런타임 쿼리 0
  • 인프라 0: 우리가 이미 가진 VERCEL_DEPLOY_HOOK_URL + node-cron 재활용
  • DEV 미리보기: 작성·검수 시 모든 글 보임 (NODE_ENV=development)
  • 스테이징: NEXT_PUBLIC_BLOG_SHOW_DRAFTS=true로 PROD에서도 미리보기
스케줄러 동작 시간

매일 09:00 KST에 Railway cron이 Vercel Deploy Hook을 호출 → 약 5~10분 빌드 → 09:10 KST 경 그날자 글 라이브.

예: 5/4 발행 예정 \"대동법\" 글은 5/4 09:00 KST cron 발동 → 09:10 경 라이브.

06 노트 ↔ 블로그 cross-link

사이트 내부 링크가 풍부할수록 SEO 강함. 양방향 자동 cross-link 시스템.

노트 → 블로그

노트 페이지 (예: /notes/s3-09) 상단에 "📖 더 자세한 가이드 →" 카드 자동 노출. getBlogPostByRelatedNoteId(noteId)가 매칭되는 블로그 포스트를 찾아 표시.

블로그 → 노트 + 학습

블로그 글 끝에 두 카드:

  • "빠른 복습 →" 카드 (왼쪽): 매핑 노트로 이동
  • "관련 기출 풀기 →" 카드 (오른쪽): /study/custom으로 이동 + era 필터

블로그 → 다른 블로그

각 글 하단 "관련 글" 섹션. getRelatedPosts(slug)가 같은 시대 + 키워드 겹치는 글을 자동 추천.

예시 — 병자호란 글에서 자동 cross-link

병자호란 (s3-09 매핑) → \"빠른 복습\" → /notes/s3-09 (광해군 중립외교와 호란)

병자호란 → \"관련 글\" → 광해군 업적, 임진왜란 결과 (같은 s3 + 키워드 겹침)

/notes/s3-09 → \"📖 더 자세한 가이드\" → /blog/byeongja-horan
