# 한국사능력검정시험 웹사이트 수익화 개발 가이드

> Claude Code로 실제 개발할 때 참고하는 문서.
> Next.js SSG 웹사이트를 만들어 **네이버 SEO + 애드센스**로 수익화한다.

---

## 1. 프로젝트 개요

### 목표
- 기존 한국사능력검정시험 학습 앱 콘텐츠를 **SEO 최적화된 웹사이트**로 배포
- **무료 서비스** (회원가입 없음) → 네이버/구글 트래픽 → 애드센스 광고 수익
- 오답노트는 localStorage로 저장 (로그인 불필요)

### 핵심 전략
- **3탭 구조**: 기출문제 / 요약노트 / 오답노트
- **광고는 콘텐츠 소비 시에만**: 해설 확인 후 + 요약노트 페이지
- **문제 풀이 중 광고 없음** → UX 우선 → 체류시간 증가
- 기출문제 ↔ 요약노트를 키워드로 양방향 연결 → 세션당 PV 극대화
- 오답노트 재방문 유도 → DAU 안정화

### 경쟁 환경 & 차별화
- 경쟁사 대부분: 문제만 있거나(기출 사이트), 정리만 있거나(블로그), 영상 위주(인강)
- **우리의 강점**: 요약노트 ↔ 기출문제가 키워드 태그로 양방향 연결
- "이 개념이 어떤 시험에서 어떻게 출제됐는지" 한눈에 보여주는 구조
- 오답노트로 개인화된 학습 경험 제공 (로그인 없이)

- 상단 헤더: 로고 + 3개 메뉴 (기출문제, 요약노트, 오답노트)

## 6. 오답노트 설계

### 핵심 특징
- **회원가입 없음** — localStorage에 저장
- **자동 저장** — 문제를 틀리면 즉시 오답노트에 추가
- **자동 해결** — 다시 풀어서 맞추면 "해결됨" 처리
- **광고 없음** — 오답 목록 자체에는 광고 미배치

### 데이터 모델

```typescript
interface WrongAnswer {
  questionId: number;
  examId: number;
  selectedAnswer: number;    // 사용자가 선택한 오답
  correctAnswer: number;     // 정답
  createdAt: string;         // 틀린 날짜 (ISO)
  resolved: boolean;         // 다시 풀어서 맞춤
  resolvedAt?: string;       // 해결 날짜 (ISO)
}

// localStorage key: "wrong-answers"
// JSON.stringify(WrongAnswer[])
```

### 자동 해결 로직

```typescript
function handleAnswerSubmit(questionId: number, selectedAnswer: number, correctAnswer: number) {
  const wrongAnswers = getWrongAnswers(); // localStorage에서 로드
  const existing = wrongAnswers.find(w => w.questionId === questionId);

  if (selectedAnswer === correctAnswer) {
    // 정답 맞춤
    if (existing && !existing.resolved) {
      existing.resolved = true;
      existing.resolvedAt = new Date().toISOString();
      saveWrongAnswers(wrongAnswers);
    }
  } else {
    // 오답
    if (!existing) {
      wrongAnswers.push({
        questionId, examId, selectedAnswer, correctAnswer,
        createdAt: new Date().toISOString(),
        resolved: false,
      });
    } else {
      existing.selectedAnswer = selectedAnswer; // 최신 오답 갱신
      existing.resolved = false;
      existing.resolvedAt = undefined;
    }
    saveWrongAnswers(wrongAnswers);
  }
}
```

## 8. 사용자 동선 & PV 순환 구조

### 검색 유입 시나리오

```
[시나리오 A: 기출문제 검색]
네이버 "한국사 45회 23번"
  → /exam/45/23 (문제 풀기)             PV 1
  → 정답 확인 + 해설 (광고 1)
  → "관련 노트: 동학농민운동" 클릭        PV 2 (광고 2)
  → "관련 기출 제52회 18번" 클릭         PV 3
  → 또 풀기 → 해설 (광고 1) → ...반복

[시나리오 B: 학습 키워드 검색]
네이버 "동학농민운동 한국사 정리"
  → /notes/donghak-movement (광고 2)   PV 1
  → "관련 기출 제45회 23번" 클릭         PV 2
  → 정답 확인 → 해설 (광고 1)
  → "같은 키워드 기출" 클릭              PV 3
  → ...반복

[시나리오 C: 오답 복습 재방문]
오답노트 방문                            PV 1
  → "다시 풀기" 클릭                     PV 2
  → 맞추면 자동 해결 + 해설 (광고 1)
  → "이 개념 복습" → 요약노트 (광고 2)   PV 3
  → ...반복
```

### 재방문 패턴
- 시험 준비 기간: 2~3개월
- 매일 접속: 기출 풀기 → 오답 복습 → 노트 확인
- 오답노트가 재방문 핵심 동력 (매일 와서 미해결 오답 줄여나감)
- 한 유저당 예상 총 PV: 300~1,000+ (준비 기간 전체)

## 9. SEO 전략 (네이버 우선)

### 네이버 SEO 필수 작업
- **SSG 필수**: 네이버 봇은 JS 실행 안 함 → HTML 완성본 필요

## 15. 주의사항

### 저작권
- 한국사능력검정시험 기출문제는 국사편찬위원회 저작물
- 교육 목적 활용은 일반적으로 허용되나, 상업적 이용 시 확인 필요
- 문제 원문 그대로 게시보다는 해설/분석 형태가 안전
- 이미지(지문 사진 등)는 별도 저작권 확인

### 기술적
- SSG 빌드 시간: 3,400+ 페이지 → 빌드 최적화 필요 (ISR 검토)
- 이미지 최적화: next/image + R2 CDN 조합
- Core Web Vitals: CLS (광고 높이 고정), LCP (이미지 lazy loading)
- 모바일 퍼스트: 사용자 대부분이 모바일 접속 예상
- localStorage 용량: 오답 1,000개 기준 약 100KB → 충분
