## Experiment Details

### E1: 경쟁앱 리뷰 분석 + 사용자 설문

**가설**: 한국사 시험 응시자의 60% 이상이 기존 학습 앱에 불만족하고 있다

**실행 방법**:
1. 앱스토어/구글플레이에서 상위 5개 한국사 앱의 1~2점 리뷰 200개 수집
2. 불만 유형 분류 (UI/UX, 콘텐츠 부족, 광고 과다, 업데이트 미비 등)
3. 네이버 카페(한국사능력검정시험), 에브리타임에 설문 배포
4. 설문 항목: 현재 사용 앱, 만족도(5점), 불만 사항, 원하는 기능

**측정 지표**: 불만 패턴 수, 불만족 응답 비율
**의사결정**:
- 성공 → 발견된 불만을 핵심 차별화 포인트로 활용
- 실패 → 차별화 전략 근본 재검토, 기존 앱과 다른 접근법 모색

---

### E2: 기출문제 저작권 법적 조사

**가설**: 한국사능력검정시험 기출문제를 앱에서 상업적으로 활용할 수 있다

**실행 방법**:
1. 국사편찬위원회 홈페이지에서 기출문제 저작권 정책 확인
2. 공공누리 유형 확인 (제1유형: 출처표시 / 제4유형: 상업적 불가)
3. 기존 한국사 앱(에듀윌, 시대고시 등)의 기출 활용 방식 조사
4. 불확실한 경우 국사편찬위원회 담당 부서에 이메일/전화 문의

**측정 지표**: 법적 활용 가능 여부 (Yes/No)
**의사결정**:
- 성공 → 기출 기반 CBT를 핵심 기능으로 확정
- 실패 → 자체 문제 제작 비용 산정, 비즈니스 모델 재검토

---

### E3: 시장 규모 & 결제 의향 조사

**가설**: 연간 응시자 50만 명 중 10%가 월 3,000~5,000원을 지불할 의향이 있다

**실행 방법**:
1. 국사편찬위원회 공개 통계에서 연간 응시자 수/회차별 추이 확인
2. 설문에 Van Westendorp 가격 민감도 질문 포함:
   - "이 앱에 얼마까지 지불하겠는가?"
   - "이 가격이면 너무 싸서 품질이 의심되는 가격은?"
   - "이 가격이면 비싸지만 고려할 가격은?"
   - "이 가격이면 절대 안 살 가격은?"
3. 경쟁앱 매출 추정 (앱스토어 순위, 리뷰 수 대비 다운로드 추정)

**측정 지표**: 연간 응시자 수, 유료 전환 의향 비율, 적정 가격대
**의사결정**:
- 성공 → 프리미엄 구독 모델 설계
- 실패 → 광고 기반 무료 모델 또는 B2B (학원/학교 대상) 검토

---

### E4: 맞춤형 분석 기능 니즈 검증

**가설**: 응시자의 70%가 "취약 영역 분석" 기능을 상위 3 원하는 기능으로 꼽는다

**실행 방법**:
1. 설문에 10개 기능 카드 제시 (랜덤 순서):
   - CBT 모의고사 / AI 오답노트 / 타임라인 맵 / 순서 맞추기 게임
   - AI 해설 / 오프라인 모드 / 학습 로드맵 / 사진문제 / 알림 / 커뮤니티
2. "가장 원하는 기능 3개를 선택해주세요" (순위 선택)
3. 추가 자유응답: "현재 학습에서 가장 불편한 점은?"

**측정 지표**: 기능별 선택 비율 및 순위
**의사결정**:
- 성공 → 오답노트/취약점 분석을 핵심 차별화 기능으로 MVP에 포함
- 실패 → 사용자가 가장 원하는 기능으로 MVP 재구성

---

### E5: AI 해설 정확도 & 신뢰도 프로토타입 테스트

**가설**: AI 생성 해설이 전문가 해설 대비 정확도 90% 이상, 사용자 신뢰도 80% 이상

**실행 방법**:
1. 기출문제 50문항 선정 (시대별/난이도별 균등 배분)
2. Claude/GPT API로 각 문항의 정답 해설 + 오답 선지별 해설 생성
3. 한국사 전공자 1~2명이 정확도 평가:
   - 정확 / 부분 정확(사소한 오류) / 오류(사실 왜곡)
4. 사용자 5명에게 블라인드 테스트:
   - AI 해설 vs 기존 해설집 해설 → "어느 쪽이 더 이해하기 쉬운가?"
5. API 비용 측정: 문항당 평균 비용 산출

**측정 지표**: 전문가 정확도 점수, 사용자 선호도, 문항당 API 비용
**의사결정**:
- 성공 → AI 해설을 프리미엄 기능으로 포함
- 부분 성공 (80-90%) → 전문가 감수 프로세스 추가 후 포함
- 실패 (<80%) → AI 해설 기능 제거, 기존 해설 활용

## Decision Framework

```
E2 (저작권) 성공
├── E1 (불만족) 성공
│   ├── E3 (시장) 성공 → ✅ 풀 스케일 MVP 개발 시작
│   └── E3 (시장) 실패 → 🔄 무료+광고 모델로 전환, 니치 타겟팅
├── E1 (불만족) 실패
│   └── 🔄 기존 앱과 다른 접근법 (예: AI 특화) 모색
│
E2 (저작권) 실패
├── 자체 문제 제작 비용 산정
├── 비용 감당 가능 → 제한적 MVP (자체 문제 + AI 해설)
└── 비용 감당 불가 → ❌ 프로젝트 피봇 또는 중단

E5 (AI 해설) 결과에 따른 MVP 범위 조정:
├── 정확도 90%+ → MVP에 AI 해설 포함 (프리미엄 기능)
├── 정확도 80-90% → 전문가 감수 추가 후 포함
└── 정확도 <80% → AI 해설 제외, CBT + 오답노트에 집중

E4 (기능 니즈) 결과에 따른 MVP 기능 구성:
├── 분석 기능 Top 3 → CBT + 오답노트/취약점 분석 + 1개 추가 기능
└── 분석 기능 Top 3 아님 → 사용자가 뽑은 Top 3 기능으로 MVP 재구성
```
