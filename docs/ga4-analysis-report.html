04 · Notes SEO Audit

노트 SEO, 지금 어디쯤인가.

/notes 랜딩 참여율 92%가 말하듯 콘텐츠는 이미 검색 유입에 강합니다. 하지만 구조화 데이터(JSON-LD)가 Breadcrumb 하나뿐이라 구글이 "이 페이지가 학습 자료임"을 명시적으로 모름. 이번 커밋으로 Article + VideoObject schema를 추가해 보강했습니다.

✓ 이미 되어 있는 것
  • ✓
    제목/설명 메타데이터 — {노트명} 요약정리 - 한능검 필수 암기 패턴
  • ✓
    Canonical URL (alternates)
  • ✓
    OpenGraph + 정적 OG 이미지 (opengraph-image.tsx)
  • ✓
    BreadcrumbList JSON-LD
  • ✓

  • ✓

  • ✓
    시대 뱃지 + 관련 기출 수 노출 + 이전/다음 노트 내부링크
△ 이번에 추가한 것

  • +
    VideoObject JSON-LD — 강의 iframe 각각에 대해 thumbnail/embedUrl/duration. 구글 영상 검색 & 리치 결과 노출 가능
  • +
    keywords meta — 노트별 10개 키워드 (한능검, 시대, 주제 변형)

05 · YouTube Tracking

이제 영상 클릭도 숫자로 본다.

지난 30일 GA4에서 video_start 이벤트 0건이었습니다. 방문자가 진짜 영상을 안 본 게 아니라, iframe에 측정 훅이 없어 GA4가 수집할 수 없었던 것. 이번에 공용 YouTubeEmbed 컴포넌트를 만들고 문제풀이 + 노트 두 곳 iframe을 교체했습니다.

작동 방식
  • 1
    iframe에 enablejsapi=1 파라미터 추가
  • 2
    iframe load 직후 postMessage({event:"listening"}) 전송 → YouTube가 상태 변화를 broadcast 시작
  • 3
    window message 리스너에서 onStateChange 수신 — state 1=playing, 0=ended
  • 4
    state 1 최초 1회 → gtag('event','video_play',…), state 0 → video_complete
  • 5
    context로 surface(question/note), question_id, exam_id, note_id, section_id 등 함께 전송
전송되는 이벤트 예시
// 문제풀이에서 영상 해설 재생 시
gtag('event', 'video_play', {
  video_id: 'abc123',
  video_title: '영상 해설',
  surface: 'question',
  question_id: 12345,
  exam_id: 77,
  question_number: 4,
  channel: '큰별쌤 최태성 1TV',
});

// 요약노트 페이지 강의 재생 시
gtag('event', 'video_play', {
  video_id: 'xyz456',
  video_title: '고려의 성립과 발전',
  surface: 'note',
  note_id: 's4-02',
  section_id: 's4',
  lecture_index: 0,
});
GA4에서 검증하는 방법 (배포 후 10~30분)
  1. GA4 → 보고서 → 참여도 → 이벤트 → video_play 행이 새로 등장하는지 확인
  2. 실시간 보고서에서 테스트 — 배포된 페이지에서 영상 재생 버튼 누르면 30초 내 이벤트 카운트 증가
  3. Debug View (?gtm_debug=x) 에서 파라미터 전부 들어오는지 확인
  4. 2주 후: video_play 수 / 문제풀이 PV 로 영상 해설 사용률 계산 → 영상 데이터 투자 ROI 판단
