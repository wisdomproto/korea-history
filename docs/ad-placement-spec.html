01 — Overview전체 구조 한눈에

02 — Provider LogicProvider 결정 로직

'auto' 모드 (default)는 디바이스 + ENV 활성화 + ID 존재 여부를 모두 보고 자동 결정. SSR-safe (useEffect mount 후 결정).

'auto' 모드 결정 트리

Step 1 Mount 후 디바이스 감지
window.innerWidth < 768
→
Step 2 모바일이면 AdFit 우선
데스크톱이면 AdSense 우선
→
Step 3 ENV 활성화 + ID 있는지 확인
없으면 다른 provider fallback
→
Step 4 둘 다 없으면 null
(광고 영역 안 보임)

3가지 호출 모드

Provider 동작 사용 예시
provider="auto"
(default)		 모바일 → AdFit / 데스크톱 → AdSense / fallback 자동 대부분 케이스. 디바이스 분리 자동 처리
provider="adsense" AdSense만 (디바이스 무관) AdSense 전용 위치 (예: 데스크톱 사이드바)
provider="adfit" AdFit만 (디바이스 무관) AdFit 전용 위치 (예: 모바일 하단 banner)
왜 디바이스 분리?

한 페이지에 두 provider 동시 노출은 광고 도배감 + AdSense 정책 가까운 광고 금지에 걸릴 수 있음. 디바이스 분리로 사용자별 한 provider만 보임 → 깨끗.

04 — Protected광고 안 띄우는 곳 (학습 보호)

정체성 보호 + 학습 핵심 가치 보호. v3 §01 Core Values "(2) 광고가 학습을 방해하면 안 된다" 원칙.

페이지 왜 광고 X 대신 해야 할 것
/study/session 학습 진행 중 — 집중 보호 광고 X. 진행 바 + 시간 표시만
/wrong-answers 오답 복습 — 핵심 가치 페이지 광고 X. retention 자산
/my-record 개인 기록 — 사용자 자산 광고 X. 약점 분석 강화로 retention
/admin/* 관리자 페이지 광고 X. 의미 없음
/board/{kind}/write 글 작성 집중 보호 광고 X. 글 작성 후 상세 페이지에서 노출
원칙

학습 진행 중 / 핵심 가치 페이지 / 작성 페이지는 광고 절대 X. 학습 끝난 후 / 콘텐츠 둘러본 후 / 결과 확인 후 = OK. 광고가 사용자 행동을 방해하면 retention 손상 → LTV 하락 → 광고 매출보다 큰 손해.

05 — ComponentAdSlot 컴포넌트 사용법

파일: web/components/AdSlot.tsx. 단일 컴포넌트로 두 provider 모두 처리. import 후 props만 정의.

기본 사용 (auto 모드, 권장)

import AdSlot from "@/components/AdSlot";

// 'auto' = 모바일 AdFit / 데스크톱 AdSense / fallback 자동
<AdSlot
  size="rectangle"
  slot={process.env.NEXT_PUBLIC_AD_SLOT_LANDING}
  className="my-6"
/>

Provider 명시 (특수 케이스)

// AdSense만 (예: 데스크톱 전용 사이드바)
<AdSlot
  size="skyscraper"
  provider="adsense"
  adsenseSlot={process.env.NEXT_PUBLIC_AD_SLOT_SIDE}
/>

// AdFit만 (예: 모바일 하단 banner)
<AdSlot
  size="banner"
  provider="adfit"
  adfitUnit={process.env.NEXT_PUBLIC_ADFIT_UNIT_BANNER}
/>

Props 정리

Prop Type Required 설명
sizeAdSize✅banner / rectangle / leaderboard / skyscraper / large-skyscraper
provider'adsense' | 'adfit' | 'auto'기본 'auto'provider 명시. 'auto'가 권장
adsenseSlotstring—AdSense slot ID. 미제공 시 slot으로 fallback
adfitUnitstring—AdFit unit ID. 미제공 시 size별 ENV 기본값 사용
slotstringdeprecatedbackward compat용 (AdSense slot)
classNamestring—wrapper div의 추가 className

06 — Env환경변수 + 활성화 절차

위치: web/.env.local (로컬) + Vercel Dashboard → Settings → Environment Variables (배포).

환경변수 spec

# ─── AdSense (심사 통과 후 활성화) ───
NEXT_PUBLIC_ADSENSE_ENABLED=false
NEXT_PUBLIC_ADSENSE_CLIENT=ca-pub-6002059361162458

NEXT_PUBLIC_AD_SLOT_QUESTION=     # 정답 확인 후
NEXT_PUBLIC_AD_SLOT_NOTE=         # 노트 강의 후
NEXT_PUBLIC_AD_SLOT_NOTE_BOTTOM=  # 노트 끝
NEXT_PUBLIC_AD_SLOT_RESULT=       # 시험 결과
NEXT_PUBLIC_AD_SLOT_LANDING=      # 랜딩

# ─── AdFit 카카오 애드핏 (통과 후 활성화) ───
NEXT_PUBLIC_ADFIT_ENABLED=false

NEXT_PUBLIC_ADFIT_UNIT_RECTANGLE=DAN-xxxxxxxxxx  # 300×250
NEXT_PUBLIC_ADFIT_UNIT_BANNER=DAN-xxxxxxxxxx     # 320×100
NEXT_PUBLIC_ADFIT_UNIT_SIDE=DAN-xxxxxxxxxx       # 160×600 / 300×600

카카오 애드핏 통과 시 절차

1. 광고 단위 생성

  1. adfit.kakao.com 로그인
  2. 매체관리 → 광고단위 추가
  3. 3개 단위 생성: rectangle (300×250), banner (320×100), side (160×600)
  4. 각 DAN-xxxxxxxxxx ID 복사

2. ENV 적용

  1. web/.env.local 열기
  2. NEXT_PUBLIC_ADFIT_ENABLED=true
  3. 3개 unit ID 채우기
  4. Vercel dashboard에도 동일 적용
  5. dev server 재시작 / Vercel redeploy

AdSense 통과 시 절차

1. 광고 단위 생성

  1. AdSense → 광고 → 광고 단위 → 광고 단위 만들기
  2. 5개 단위 생성 (모두 rectangle 형):
    • QUESTION (정답 확인)
    • NOTE (노트 강의 후)
    • NOTE_BOTTOM (노트 끝)
    • RESULT (시험 결과)
    • LANDING (랜딩)
  3. 각 slot ID 복사

2. ENV 적용

  1. web/.env.local 열기
  2. NEXT_PUBLIC_ADSENSE_ENABLED=true
  3. 5개 slot ID 채우기
  4. Vercel dashboard에도 동일 적용
  5. dev server 재시작 / Vercel redeploy
검증

활성화 후 5곳 페이지 (랜딩 / 문제풀이 / 노트 / 결과)에서 광고가 보이는지 확인. 모바일 viewport (devtools) → AdFit 표시 / 데스크톱 → AdSense 표시. 둘 다 동시 보이면 X — provider 분리 작동 확인.

광고 통과 후 또는 트래픽 도달 후 추가할 위치들. 우선순위 표시.

위치 사이즈 우선순위 트리거 조건 비고
/board (hub) banner P2 광고 통과 후 탭 아래 가벼운 노출. 사용자 결정 (4/26): "나중에 하자"

08 — Rules광고 디자인 원칙

v3 마케팅 전략 §01 Core Values + §07 Counter Metrics 기반.

✅ DO

  • 학습 끝난 직후 또는 결과 확인 시점에 노출
  • 한 페이지에 광고 ≤ 2개 (도배 X)
  • 디바이스 분리 (모바일 AdFit / 데스크톱 AdSense)
  • Above the fold X (Hero 첫 화면엔 광고 X)
  • 광고와 콘텐츠 사이 충분한 여백 (my-6 이상)
  • 학습 흐름 핵심 페이지는 명시적으로 광고 X

❌ DON'T

  • 학습 진행 중 (/study/session) 광고 노출
  • 오답노트 / 내 기록 등 retention 자산 페이지에 광고
  • 한 페이지에 같은 provider 광고 3개+
  • Pop-up / Interstitial / 자동 재생 광고 (절대 X)
  • 콘텐츠와 광고 구분 안 되는 디자인 (AdSense 정책 위반)
  • "광고 도배 없이"라는 카피 사용 (실제 운영과 모순)

Counter Metric — 광고 도배 감지

v3 §07 Measurement Framework의 counter metrics 중 광고 관련:

지표 경계 (Watch) 위험 (Warn) 의미
광고비 ÷ 매출 비율≥ 30%≥ 50%광고 의존 — 오가닉 약화 신호
세션당 페이지≤ 4≤ 2광고 때문에 사용자 이탈 가능
Direct 비율≤ 30%≤ 15%retention 손상 — 광고가 원인일 수 있음
광고 클릭률 (CTR)> 2%> 5%실수 클릭 의심 — 광고 위치 점검

한 지표라도 'Warn' 진입 시 즉시 광고 위치 stop & review.
